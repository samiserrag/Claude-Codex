Design notes (mockup-only — what changed v2 → v3)

Folded in three threads from the convergence conversation plus Sami's role-agnostic caveat. Same visual palette and typography as v2.

  • Role-agnostic UI vocabulary (Sami's caveat — top priority). All UI text uses capability/role labels: Coordinator, Builder, Auditor, Human approver. Model names appear only in the "Current assignments" metadata block at the top, marked explicitly as "local and swappable." Replaces v2's "paste to Claude Code" / "Ask GPT" / "Claude auditor" / "GPT exit synthesis" everywhere.
  • Decision options block (Codex's addition). 4 alternatives as compact rows by default. Each expands via native <details> for pros / cons / "does NOT authorize." Solves density without hiding alternatives. "Reject" intentionally omitted — not yet in protocol vocabulary; "Request revision" covers the legitimate decline case.
  • Risk context as multi-part breakdown (GPT's addition). Consequence / Evidence confidence / Reversibility / Open caveats / Unknowns / If wrong / Human approval required. Disclaimer "This summarizes what the agents found. It does not decide for you." Prose required for caveats, unknowns, and "if wrong" — prevents label-only decay. No single risk badge anywhere.
  • "Primary action surfaced" replaces "Recommended" framing. The system surfaces a default; it does not recommend approval. Removes a sycophantic tilt.
  • Approval boundary refactored into two-column "DOES / does NOT" grid. Pairs naturally; replaces v2's single one-sided list.
  • Same v2 visual palette and discipline: sage/amber/copper accents, calm dark surfaces, generous breathing room, no progress rings for approval states, no vanity metrics, no drag-drop.

Pushbacks I'd flag for future iteration:

  • Risk context could decay to label-only if agents auto-fill the prose fields with boilerplate. Counter: occasional human review of "is this risk context actually useful, or has it become noise?"
  • Decision options expandable adds latent density. If most users never expand, the pros/cons are wasted; if everyone expands them, the screen gets noisy. Worth testing.
  • "Current assignments" block at top is a small product statement — saying "we name roles, not models" with the layout. Could be moved to a footer if it pulls focus from the role row, but I think top-placement is right for new users learning the model-agnostic principle.
